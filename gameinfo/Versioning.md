## Here is how versioning will work from now on in Roboquix. 
## Based a bit on, but not truly, Semantic Versioning.

**Major:** This will most likely always stay at 1, unless:
  - The progression system for the game has been completely reworked
  - The original core of the game has become difficult to maintain
  - Save management is reworked

**Minor:** From now on, after 1.3, this value will be incremented if at least 2 of, but not limited to, these features are added:
  - One new world with at least one unique mechanic
  - A few changes to the UI
  - A new character
  - Other new content (New Achievements, QoL improvements, etc.)
  - Patch version has reached 9 (May only apply after 1.4)

**Patch:** This will be incremented if:
  - A set of bugs have been fixed
  - A small improvement has been made
  - Will not be incremented upon new features unless if something critical is added!

**Build:** This represents the true existing build of the game (4 digits). Changes which could occur:
  - Incremented every time I push any change of the game.
  - Reset to 0000 if major is incremented.
